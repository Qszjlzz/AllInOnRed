# 团队分工 — Game Jam 四人组

**项目**：别按那个键  
**周期建议**：Day1 对齐 → Day2–3 并行 → Day4 联调 → Day5 打磨提交

---

## 角色一览

| 角色 | 负责人 | 核心产出 |
|------|--------|----------|
| 文案 | TBD | 全部中文台词、事件表、结局文本 |
| 美术 | TBD | 壁纸、图标、UI 皮肤、女儿画作 |
| 程序 | TBD | HTML Demo 可玩版本 + 系统架构 |
| 数值策划 | TBD | 随机范围、阈值、结局条件表 |

---

## 1. 文案（Writer）

### 负责
- 五应用内全部可见文本（聊天、通知、按钮、结局）
- `WRITER_BRIEF.md` 事件清单填写与扩写
- 语气指南执行：疲惫、克制、逐渐失控
- Day3 备忘录引导句 + climax 回放衔接
- 结局标题与结尾旁白（4+ 条）

### 不负责
- 程序逻辑、数值具体数字（与数值策划对齐后定稿）
- 美术资源制作

### 交付物
- `docs/events_copy.md`（可选，或在 WRITER_BRIEF 内标注完成）
- 字符串表（CSV/JSON，供程序 `import`）

### 依赖
- **输入**：DEMO_PLAN 故事节拍
- **输出给**：程序（字符串 key）、美术（画作描述供出图）

---

## 2. 美术（Art）

### 负责
- `ART_BRIEF.md` 所列每一项资产
- 统一视觉：深夜办公室、冷色桌面、低饱和
- 应用图标、壁纸、窗口装饰、女儿画作、转盘背景
- 为每项资产提供 **AI 生图 prompt**（已在 ART_BRIEF 预填，可调整）

### 不负责
- 程序交互逻辑
- 文案定稿

### 交付物
- `assets/` 目录下 PNG/WebP（命名与 ART_BRIEF 一致）
- 导出规格说明（@1x，透明底 where needed）

### 依赖
- **输入**：文案（画作主题文字）、程序（尺寸）
- **输出给**：程序（替换 CSS 占位）

---

## 3. 程序（Programmer）

### 负责
- `index.html` + `css/` + `js/` 完整 Demo
- 桌面壳、窗口管理、游戏状态机
- 事件系统、赌博、工作 QTE、结局判定
- 与数值策划对接：读取 `balance.json` 或内联常量
- `file://` 兼容，无构建步骤

### 不负责
- 最终文案润色（接字符串表）
- 原画绘制

### 交付物
- 可运行 Demo 根目录
- `PROGRAMMER_BRIEF.md` 架构实现
- 注释标明 `// ART: asset_name` 占位位

### 依赖
- **输入**：文案字符串、美术 assets、数值 balance 表
- **输出给**：全员试玩包

---

## 4. 数值策划（Balance）

### 负责
- `BALANCE_BRIEF.md` 全部范围与权重
- 工作收入、赌博盈亏、账单目标
- mood 阈值、上瘾强制事件触发
- 结局条件矩阵（债务 × 按键 × 早期止损）
- Playtest 记录 3 轮，调整区间

### 不负责
- 代码实现（提供表格式 spec）
- 叙事写作

### 交付物
- `js/balance.js` 或 `data/balance.json`
- 结局判定真值表

### 依赖
- **输入**：DEMO_PLAN 机制、程序 API 说明
- **输出给**：程序（可直接粘贴的配置）

---

## 协作时间线（5 天 Jam）

```
Day 1 上午   全员读 DEMO_PLAN，程序搭桌面壳空窗
Day 1 下午   文案出 Day1–3 事件初稿；数值出 balance v0.1
Day 2        美术出壁纸+图标；程序接事件/赌博/工作
Day 3        文案定稿；美术出画作+UI；数值 playtest v0.2
Day 4 上午   联调：换皮、换文案、调数值
Day 4 下午   全员试玩，录屏，bug 列表
Day 5        修 bug、stretch 砍 scope、打包 README
```

## 交接检查点

| 检查点 | 参与人 | 通过标准 |
|--------|--------|----------|
| CP1 桌面可点 | 程序 | 5 窗口能开闭 |
| CP2 一天能打完 | 程序+数值 | AP/金钱/账单动 |
| CP3 文案全覆盖 | 文案+程序 | 无 lorem ipsum |
| CP4 视觉统一 | 美术+程序 | 占位全部替换或标记 TODO |
| CP5 结局可达 | 全员 | 4 结局各至少通一次 |
