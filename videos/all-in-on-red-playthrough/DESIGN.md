# Design System

## Overview

《红键梭哈》是一套像素桌面叙事界面，主画面建立在“深夜办公室 + 雨夜窗景 + 旧显示器冷光”之上。整体布局不是网页滚屏，而是桌面操作系统式的多窗口关系：顶部 HUD、中央工作区、底部剧情栏、任务栏式应用入口。色彩策略非常明确，深紫黑做大面积背景，青蓝负责标题与系统感，红色专门留给“危险按钮”和上瘾提示，暖黄只在灯光和警示信息里出现。整体气质是克制的、压抑的、复古的，但所有关键反馈又必须足够直白。

## Colors

- **Primary Surface**: `#1A1423` — 桌面主背景，深夜基底
- **Panel Surface**: `#2D2640` — 面板与按钮常规底色
- **Window Surface**: `#3D3555` — 窗口主体层
- **Window Title**: `#4A4068` — 标题条与高一层 UI
- **Primary Text**: `#E8E0F0` — 正文与 HUD 文本
- **Muted Text**: `#9A8FB0` — 次级提示与说明
- **System Accent**: `#6EC6FF` — 标题、可操作高亮、赛博感冷色
- **System Accent Dim**: `#4A8FBF` — 次级强调按钮
- **Danger Accent**: `#FF4466` — 红键、风险提示、沉迷主题中心色
- **Warning Accent**: `#FFCC44` — 警示、关键说明、暖色对比
- **Narrative Panel**: `#1E1830` — 剧情栏底色
- **Border**: `#6A5A8A` — 统一像素边框

## Typography

- **Display**: `Press Start 2P` (400, 700) — 中文标题、英文字标、像素品牌感
- **UI / Body**: `Microsoft YaHei` (400, 600, 700) — 正文、字幕、HUD、按钮
- **Fallback**: `Arial` (400) — 仅作系统后备，不作为风格主体
- **Hierarchy**: 主标题使用小尺寸像素字但高辨识度冷青色；正文保持 12px 左右密度；按钮和 HUD 维持 10px-12px 的桌面软件感

## Elevation

这套界面的“层级感”不靠圆角和毛玻璃，而靠像素描边、块面分层和少量黑色阴影完成。按钮、窗口、任务栏全部使用硬边矩形与 2px 边框，阴影只做一层 `rgba(0, 0, 0, 0.5) 6px 6px 0px 0px` 的旧像素 UI 质感。背景本身带有雨夜办公室插画，因此前景不再叠重阴影，而是通过亮暗对比和色温对比把焦点压到红键与字幕上。

## Components

- **Rain Office Backdrop** — 雨夜像素办公室背景，整套情绪来源
- **HUD Strip** — 顶部状态栏，显示周期、现金、债务、按键次数
- **Narrative Dock** — 底部剧情条，承载主要文字叙事
- **Pixel App Rail** — 任务栏式应用入口，控制聊天、赌博、工作、家庭窗口
- **Red Gamble Panel** — 红键窗口，中央按钮必须是第一视觉焦点
- **Impact Delta Chips** — 每次按键后显示现金、机器、债务变化的即时反馈条
- **Cover Screen** — 开始界面，大标题、主题句、简介与开始按钮组成封面

## Do's and Don'ts

### Do's

- 使用深紫黑做大面积底色，让红键成为全片最危险的高饱和点
- 保留像素描边和硬边矩形，不把界面做成现代圆角 SaaS
- 字幕和说明统一使用清晰的浅色文字，保证录屏里一眼可读
- 用暖黄灯光、冷蓝系统光、危险红三种色温做画面节奏
- 保持“桌面正在被诱惑侵入”的感觉，窗口和提示要像系统事件

### Don'ts

- 不要改成纯黑极简或霓虹夜店风，那会丢掉办公室叙事感
- 不要使用大面积模糊玻璃、发光描边海量堆叠或现代卡片圆角
- 不要让字幕颜色过冷或过灰，录屏压缩后会立刻失去可读性
- 不要把红色随便分配给别的元素，危险感必须留给按钮和关键警示
- 不要把画面做得太空，桌面视频需要保持 HUD、剧情和窗口的存在感
